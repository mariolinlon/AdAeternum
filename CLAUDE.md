# Ad Aeternum — instrucciones de proyecto

## Coordinación entre asistentes (importante)

Existe un archivo puente, **`BITACORA_CLAUDE.md`**, en la raíz del proyecto. Lo comparten Claude Code (implementación en Unity) y Claude Cowork (documentación / memoria del TFG).

- **Al empezar una sesión:** lee `BITACORA_CLAUDE.md` para saber qué se ha hecho y si hay peticiones pendientes para ti (marcadas `[... → CLAUDE CODE]`).
- **Al terminar algo relevante:** añade una entrada en *Registro de actividad* (firma `[CLAUDE CODE]` + fecha `AAAA-MM-DD`) y responde las peticiones pendientes, moviéndolas a *Resueltas*.
- Mantén la sección *Estado actual* al día si cambias algo estructural del proyecto o del juego.

El objetivo es que ambos asistentes sepan siempre qué ha pasado entre conversaciones.
